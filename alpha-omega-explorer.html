Alpha / Omega scenario explorer

Block reward disposition mechanism — α (revenue-gated) + ω (time-based) + burn (ω − α)

How it works
ω Omega goes up on a fixed time schedule — total % of block rewards removed from consensus. No revenue gate.
α Alpha is revenue-gated — only steps up when deal revenue clears the target. Controls how much goes to the service treasury.
burn ω − α = burned. If α tracks ω, minimal burn (service economy funded). If α lags, the gap burns (deflationary).

Alpha gate formula
gate = beta × BR_q × alpha  |  if R_q ≥ gate → alpha += step_size, else hold
beta = beta_start + (beta_max − beta_start) × ((alpha − alpha_start) / (alpha_max − alpha_start))k
More info: FIL+ Reform – Alpha Ramp-Up Design Doc
How to use this explorer
Step 1: Choose a revenue growth model below. Deal revenue = total $ value of paid storage deals (not the 0.5% network fee). Adjust the starting revenue, growth rate, and FIL price to match your assumptions.
Step 2: Set the protocol parameters (α step size, β escalation, gate shape) to see how they affect the alpha trajectory.
Step 3: Configure the omega (ω) schedule — the time-based consensus reduction that proceeds regardless of revenue. The gap ω − α is burned.

How these inputs work ↓

Starting quarterly deal revenue (R0) is the Q1 value — total $ value of paid storage deals on-chain (FOC market). Growth rate (g) compounds it forward across 16 quarters using one of two models:

Decaying growth: R(i) = R(i−1) × (1 + gi) where gi = floor + (g − floor) × e−0.45(i−1)
Growth rate starts at g (slider value) in Q2, decays exponentially toward the floor rate steady state.

Peak-and-settle: R(i) = R(i−1) × (1 + gi), gamma-shaped growth rate normalised so g1 = g (slider value)
Starts at g, ramps up over 2–3 quarters (peak ~Q4), then steep decay to floor by ~Q13. As revenue gets large, growth rate drops fast.

Wasabi benchmark: Implied QoQ growth rates derived from Wasabi Technologies (cloud storage, founded 2017). Verified annual revenue: $18M→$54M→$92.5M→$134.7M→$160.1M (2020–2024). Sources: getlatka.com, blocksandfiles.com, wasabi.com press releases.
Each year’s QoQ = (end/start)1/4−1, held constant across 4 quarters: 32%/q → 14%/q → 10%/q → 4%/q. Growth rate and floor rate sliders are ignored. Starting deal revenue slider still sets R0 (Wasabi’s absolute $ figures don’t apply to Filecoin — only the growth shape is borrowed).

Block reward: 63,727 FIL/day (Apr 2026, via Filfox) with 6-year half-life decay. BRq = start-of-quarter daily rate × 90 days × FIL price (~1.4% overestimate vs exact integration).

Note: This model only applies simple minting decay to BRq. It does not account for baseline minting, which depends on network QAP vs the baseline power function. Currently baseline contributes ~14% of daily emission because QAP is well below baseline. If QAP were to grow, baseline minting would increase BRq beyond what this model projects, raising the gate target.
