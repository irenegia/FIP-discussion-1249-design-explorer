Alpha step-up scenario explorer

Revenue-gated BR redirect mechanism — interactive parameter explorer

Design principles
P1 Don't step up without proof — revenue gates every increase
P2 Start small, step frequently — 5% initial, quarterly or continuous
P3 Revenue bar to pass the gate escalates with alpha — shape controlled by k
More info: FIL+ Reform – Alpha Ramp-Up Design Doc
How to use this explorer
Step 1: Choose a revenue growth model below. The goal is to pick the model that most accurately predicts what will happen to revenue in the Filecoin on-chain deal market (FOC). Adjust the starting revenue, growth rate, and FIL price to match your assumptions.
Step 2: Play with the protocol parameters to see how different settings affect the alpha trajectory. Use this to choose the parameter values to propose in the FIP.

How these inputs work ↓

Starting quarterly revenue is the Q1 value — current on-chain deal revenue (FOC market). Growth rate compounds it forward across 16 quarters: exponential applies the rate each quarter; linear adds a fixed $ amount (= start × rate); decelerating starts at the full rate then tapers; S-curve starts slow, accelerates, then plateaus. At each quarter the model checks whether projected revenue clears the gate (beta × treasury take) to decide if alpha steps up.

Block reward: 63,727 FIL/day (Apr 2026, via Filfox) with 6-year half-life decay. BRq = start-of-quarter daily rate × 90 days × FIL price (~1.4% overestimate vs exact integration).

Note: This model only applies simple minting decay to BRq. It does not account for baseline minting, which depends on network QAP vs the baseline power function. Currently baseline contributes ~14% of daily emission because QAP is well below baseline. If QAP were to grow, baseline minting would increase BRq beyond what this model projects, raising the gate target.
